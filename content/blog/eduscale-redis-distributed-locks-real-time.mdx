    <h2>Takeaways</h2>

    <ul>
      <li><strong>Socket.io rooms don't scale horizontally without the Redis adapter.</strong> This is obvious in retrospect but not obvious the first time you add a second server instance.</li>
      <li><strong>Distributed locks require careful TTL design.</strong> Too short and you get double-fire. Too long and a crashed lock holder blocks progress for the full TTL.</li>
      <li><strong>retryCount:0 is the right choice for game ticks.</strong> A delayed tick is worse than a skipped tick in a real-time game context.</li>
      <li><strong>Circuit breakers need realistic fallbacks.</strong> A fallback that surfaces an error is better than a fallback that silently returns wrong data.</li>
